🔹 Escenario

uso de ramas en git hub

git clone https://github.com/Lopz532/Programadores.git


Después puede crear su propia rama:

git checkout -b feature/mi_proyecto

🔹 Qué pasa ahora

Él trabajará solo en su rama (feature/mi_proyecto).

Todo lo que haga no afecta tu rama main ni tu proyecto base.

Mientras él haga commits en su rama, tú puedes seguir trabajando en main o en otras ramas sin problemas.

🔹 Cuando quiera integrar cambios

Solo si su proyecto está listo y quieren unirlo al main, se hace un merge:

git checkout main
git merge feature/mi_proyecto
git push origin main


Hasta ese momento, los cambios en su rama siguen independientes.

💡 Resumen simple:

Ramas = espacios de trabajo independientes.

Puedes clonar repos, crear ramas y trabajar sin tocar el proyecto de otros.

Solo al fusionar (merge) se combinan los cambios.
